# Project Instructions

## User Interaction

- When asking the user a question, use the Spokenly `ask_user_dictation` tool.
- Do not ask questions as plain text.

## Product Design

- Do not use eyebrow labels, kicker text, or small subtitles above main titles.
- Prefer fewer words in fewer places.
- When implementing self-directed UI work, make state, meaning, and hierarchy recognizable through clear patterns, shapes, color, spacing, progress, and motion before adding explanatory text.
- Avoid redundant labels when the same information is already communicated by position, component shape, color, or nearby content.
- When comparing design directions, vary component language and interaction patterns, not only color palettes. Show differences in buttons, lists, tabs, modals, spacing, borders, density, type scale, and feedback states.
